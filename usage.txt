============================================================
  Preprocessing project - Guia de arranque y uso
============================================================

RESUMEN RAPIDO
--------------
Tres comandos para tener todo funcionando:

  cd ~/Escritorio/WorkInProgress/Preprocessing
  docker compose build          # solo la primera vez o si cambian deps
  docker compose up -d --force-recreate  # levanta con imagenes recien construidas

Después abre:  http://127.0.0.1:8888  (el token lo ves en la sección 4)


============================================================
1. REQUISITOS PREVIOS
============================================================

- Docker con soporte GPU (nvidia-container-toolkit instalado)
- Datasets locales en:  ~/Escritorio/Datasets/
  Estructura esperada:
    ~/Escritorio/Datasets/
    ├── INbreast/AllDICOMs/       <- usado en notebooks_api/06
    └── Mammo-MX/B1/              <- alternativa
- Repositorio clonado en:  ~/Escritorio/WorkInProgress/Preprocessing/

============================================================
4. ARRANQUE COMPLETO PASO A PASO
============================================================

-- Paso 1: construir imagenes (solo si es la primera vez
           o si cambian dependencias Python/apt) --

  cd ~/Escritorio/WorkInProgress/Preprocessing
  docker compose build

  Para reconstruir solo un servicio:
    docker compose build yolox-api
    docker compose build maseg-api
    docker compose build notebook

  Arquitectura de stages en docker/Dockerfile.services:
    - base          -> librerias comunes del proyecto (requirements.txt)
    - yolox_runtime -> base + extras de MyYoloX
    - maseg_runtime -> base + extras de MAseg
    - notebook_runtime -> base + extras de MyYoloX y MAseg para notebooks
    - yolox_api     -> deriva de yolox_runtime
    - maseg_api     -> deriva de maseg_runtime
    - notebook      -> deriva de notebook_runtime

-- Paso 2: levantar servicios en background --

  docker compose up -d --force-recreate

-- Paso 3: comprobar que todo esta vivo --

  docker compose ps
  curl http://localhost:8001/health    # {"status":"ok","service":"yolox"}
  curl http://localhost:8002/health    # {"status":"ok","service":"maseg"}

-- Paso 4: obtener el token de JupyterLab --

  docker logs preprocessing-notebook 2>&1 | grep 'token='

  Abre en el navegador o en VS Code:
    http://127.0.0.1:8888/lab?token=<TOKEN>

-- Paso 5: conectar VS Code al kernel del contenedor --

  1. Abre notebooks_api/06_services_yolox_breastsegmentation.ipynb
  2. Selector de kernel -> "Existing Jupyter Server"
  3. Pega la URL con token del paso 4

-- Paso 6: ejecutar el notebook de prueba --

  notebooks_api/06_services_yolox_breastsegmentation.ipynb
  Ejecuta celdas en orden:
    Celdas 1-4 -> carga DICOM y preprocesado con api_stable.MammographyDicom
    Celdas 5-7 -> health checks + inferencia via API HTTP (YOLOX y MAseg)


============================================================
5. SERVICIOS DISPONIBLES
============================================================

  Servicio          URL                    Descripcion
  ----------------  ---------------------  -----------------------
  JupyterLab        http://localhost:8888  Notebooks interactivos
  YOLOX API         http://localhost:8001  Deteccion bounding boxes
  MAseg API         http://localhost:8002  Segmentacion pectoral

  Endpoints API:
    GET  /health  -> {"status":"ok","service":"<nombre>"}
    POST /infer   -> recibe imagen PNG/JPG como multipart/form-data
                     devuelve JSON con detecciones o mascara

  Ejemplo curl YOLOX:
    curl -s -X POST http://localhost:8001/infer \
      -F "file=@/ruta/imagen.png" \
      -F "input_format=BGR"

  Ejemplo curl MAseg:
    curl -s -X POST http://localhost:8002/infer \
      -F "file=@/ruta/imagen.png" \
      -F "fill_holes_in_breast=true" \
      -F "include_pectoral_mask=false"


============================================================
6. COMANDOS UTILITARIOS DESDE LA RAIZ
============================================================

  Exportar tags DICOM:
    python utils/extract_dicoms.py /ruta/dataset /ruta/salida/tags.csv \
      --include-extensionless \
      --workers auto

  Convertir DICOM a PNG de 8 bits:
    python utils/export_dicom_pngs.py /ruta/dataset_dicom /ruta/dataset_png \
      --include-extensionless \
      --workers auto

  Visualizar overlays imagen+mascara:
    python -m image.visualization /ruta/imagenes /ruta/mascaras \
      --batch-size 8 \
      --columns 4


============================================================
7. PARAR Y LIMPIAR
============================================================

  Parar servicios (sin borrar imagenes):
    docker compose stop

  Parar y eliminar contenedores y red:
    docker compose down

  Eliminar tambien las imagenes construidas:
    docker compose down --rmi local


============================================================
8. PROBLEMAS FRECUENTES
============================================================

- Dataset no encontrado en el notebook:
  Comprueba que ~/Escritorio/Datasets/ existe y que DICOM_PATH
  apunta a un archivo valido dentro de /workspace/Data/

- CUDA no disponible:
  Ajusta YOLOX_DEVICE=cpu en docker-compose.yml para la API.

- Puerto ya en uso:
  docker compose down  y vuelve a ejecutar  docker compose up -d --force-recreate

- Token de Jupyter ha cambiado:
    docker logs preprocessing-notebook 2>&1 | grep 'token='

- Reconstruir dependencias tras cambios en requirements:
    docker compose build <servicio>   # notebook | yolox-api | maseg-api
  docker compose up -d --force-recreate <servicio>
